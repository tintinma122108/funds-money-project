# 基金估值助手 APP - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
基金估值助手是一款专注于为用户提供实时基金持仓估值计算的工具型应用。通过实时获取基金价格数据，自动计算用户持仓的浮动盈亏、总资产及收益情况，帮助用户快速了解投资组合的实时表现。

### 1.2 核心价值
- **实时性**：每2秒自动刷新基金价格，提供准实时的估值数据
- **准确性**：基于真实基金API数据，确保计算结果的准确性
- **便捷性**：简洁的操作流程，快速添加和管理持仓基金
- **可视化**：复古工业风设计，直观展示持仓收益和涨跌情况

### 1.3 目标用户
- 持有多个基金的投资者
- 需要实时了解持仓收益的用户
- 关注基金表现的理财用户

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 基金持仓管理
**功能描述**：用户可手动添加、查看、删除持有的基金

**详细需求**：
- **添加基金（简易录入）**
  - 输入字段（仅三项）：
    - 基金代码（6位数字，必填）
    - 持仓份额（数字，必填，单位：份）
    - 持有均价（数字，必填，单位：元/份）
  - 输入验证：基金代码格式与有效性、持仓份额 > 0、持有均价 > 0
  - 计算逻辑：当前市值 = API 净值 × 份额，持有成本 = 份额 × 持有均价，持有收益 = 当前市值 - 持有成本（由服务端/API 自动完成）
  - 交互流程：
    - 点击"添加"按钮打开添加基金模态框
    - 填写基金信息
    - 支持取消操作（点击取消或关闭按钮）
    - 确认后自动获取基金名称并添加到持仓列表
  - 数据存储：使用 localStorage 持久化存储

- **删除基金**
  - 交互方式：长按基金卡片或点击删除按钮
  - 确认机制：删除前弹出确认对话框
  - 数据同步：删除后立即更新总持仓统计

- **基金列表展示**
  - 排序规则：按持有金额从大到小排列
  - 布局方式：单列垂直布局，每个基金独立卡片展示
  - 滚动支持：超出可视区域时支持滚动查看更多
  - 间距设计：遵循信息亲密度原则，保持舒适的视觉间隔

#### 2.1.2 实时估值计算
**功能描述**：实时获取基金价格并计算持仓估值

**详细需求**：
- **数据获取**
  - 数据源：通过多个免费API源获取基金数据（天天基金、新浪财经、东方财富等）
  - 容错机制：多数据源备选，API失败时使用缓存数据或默认值
  - 批量查询：支持批量获取多个基金的价格数据，提高效率
  - 缓存策略：1.5秒缓存机制，减少不必要的API请求

- **更新频率**
  - 刷新间隔：每2秒自动刷新一次
  - 更新范围：所有持仓基金的价格数据
  - 动画反馈：数据变化时使用数字动画效果提示用户

- **计算逻辑**
  - 持有成本 = 持仓份额 × 持有均价（元/份）
  - 当前市值 = API 当前净值 × 持有份额（本交易日结束后取最新公布净值）
  - 持有收益 = 当前市值 - 持有成本
  - 持有收益率 = (持有收益 / 持有成本) × 100%
  - 估算今日收益 = (当前价格 - 昨日净值) × 持有份额
  - 估算今日涨跌 = API 提供的涨跌幅或计算值

#### 2.1.3 总持仓统计
**功能描述**：汇总展示所有持仓基金的整体情况

**详细需求**：
- **统计指标**
  - 总持仓金额：所有基金的持仓市值总和（包含收益）
  - 持仓总收益：所有基金的持有收益相加
  - 估算今日总收益：所有基金的今日收益加总
  - 估算今日涨跌：基于总持仓金额计算的当日综合涨跌幅

- **展示方式**
  - 固定位置：页面顶部固定展示
  - 隐私保护：支持点击展开/收起，默认显示遮罩效果
  - 数据更新：实时同步更新，配合动画效果

#### 2.1.4 单基金信息展示
**功能描述**：在基金列表中展示每只基金的详细信息

**详细需求**：
- **展示字段**
  - 基金名称：自动从 API 获取
  - 基金代码：用户输入的 6 位代码
  - 持有金额：当前市值 = 份额 × API 当前净值（实时计算）
  - 持有份额：用户输入的持仓份额
  - 持仓成本：份额 × 用户输入的持有均价
  - 持有收益：当前市值 - 持仓成本
  - 持有收益率：(持有收益 / 持仓成本) × 100%
  - 估算今日收益：当日净值变更 × 份额
  - 估算今日涨跌：API 涨跌幅或计算值

- **视觉设计**
  - 卡片样式：白色背景，撕纸边缘效果
  - 颜色标识：收益为正显示红色，收益为负显示绿色
  - 字体规范：数字使用等宽字体（tabular-nums），标签使用常规字体

## 3. 交互设计

### 3.1 列表交互
- **排序规则**：基金按持有金额大小降序排列
- **滚动行为**：超出可视区域时支持垂直滚动
- **间距设计**：单基金卡片之间保持舒适的间隔（遵循信息亲密度原则）
- **删除操作**：
  - 触发方式：点击删除按钮
  - 确认机制：删除前弹出确认对话框
  - 动画效果：删除时使用淡出动画

### 3.2 数据刷新交互
- **刷新频率**：每2秒自动刷新一次
- **动画反馈**：
  - 数字变化时使用数字滚动动画
  - 总持仓卡片有轻微的缩放和颜色变化提示
- **加载状态**：API请求失败时静默处理，使用缓存数据或默认值

### 3.3 模态框交互
- **打开动画**：从中心缩放出现，带有轻微的上移动画
- **关闭动画**：
  - 取消操作：纸张揉皱动画效果
  - 确认操作：向下翻转消失动画

  - 收益负数：绿色系（#3a6e5a, #059669）
  - 辅助文字：灰色（#6b7280, #a8a29e）

### 4.2 字体规范
- **Slogan 字体**：Academy Engraved LET（雕刻感衬线字体，通过 @font-face 嵌入）；Slogan 文案全大写：「FUND YOUR FORTUNE」；不写死行数，视窗宽时单行、变窄时按词自动换行；字号随视窗自适应（clamp）
- **数字字体 (Digital / Tech)**：
  - 应用对象：所有数字数据，包括：
    - 总资产数字（PortfolioHeader）
    - 估值涨跌幅（FundCard）
    - 累计收益、当日盈亏（PortfolioHeader）
    - 持有金额、持有份额、持仓成本、持有收益、持有收益率（FundCard）
    - 基金代码（FundCard）
    - 输入框数字（AddFundModal）
  - 字体样式：Share Tech Mono（通过 Google Fonts 嵌入）
  - 特点：统一使用 Share Tech Mono 字体，具有强烈的科技感和仪表盘质感，模拟液晶数值风格，确保所有数字数据的一致性和专业性
  - 字体特性：所有数字使用 `tabular-nums`（等宽数字）和 `fontVariantNumeric: 'tabular-nums'` 确保数字对齐
- **标签字体**：系统默认字体
- **字体特性**：
  - 数字使用 `tabular-nums`（等宽数字）
  - 标签文字使用 `tracking-wider`（字间距加宽）
  - 数字文字使用 `tracking-normal`（正常字间距）
- **字体嵌入**：所有自定义字体通过 Next.js `next/font/google` 自动优化加载，确保不同用户访问时显示一致的字体效果

  - 最大宽度：1280px（居中显示）

- **卡片布局**：
  - 卡片内边距：32px

  - 固定位置：页面顶部
  - 内边距：24px

### 4.4 动画效果
- **数字更新动画（RollingNumber 组件）**：
  - 使用 RollingNumber 组件实现类似老虎机或里程表的垂直滚动动画
  - 每个数字位（0-9）独立滚动，根据数值变化精准滚动到目标数字
  - 支持小数位、正负号前缀和百分比后缀，确保金融数据格式（如 +12.34% 或 -500.00）在动画过程中准确
  - 应用对象：总资产数字、累计收益、当日盈亏等所有数字数据

- **总持仓卡片动画**：
  - 展开/收起时3D翻转效果

## 5. 技术实现

### 5.1 数据存储
- **存储方式**：localStorage
- **存储内容**：基金持仓列表（JSON格式）
- **同步机制**：持仓数据变更时立即保存到 localStorage

### 5.2 API集成
- **数据源**：
  - 天天基金网 API
  - 新浪财经 API
  - 东方财富 API
- **容错机制**：
  - 多数据源备选
  - API失败时使用缓存数据
  - 缓存失效时使用默认值
- **性能优化**：
  - 批量查询支持
  - 1.5秒缓存机制
  - 请求去重处理

### 5.3 实时更新机制
- **更新频率**：每2秒刷新一次
- **更新范围**：所有持仓基金
- **更新策略**：
  - 使用 ref 避免竞态条件
  - 批量获取价格数据
  - 同步更新所有持仓信息

### 5.4 计算逻辑
- **持有份额**：用户直接输入，不需要计算
- **持有成本计算**：`holdingCost = shares × purchasePrice`（根据用户输入的份额和成本价计算）
- **当前市值（列表显示）**：`currentValue = shares × currentPrice`（实时计算，每2秒更新）
- **持有金额（列表显示）**：使用 `currentValue`，即 `shares × currentPrice`（实时计算）
- **持有收益计算**：`holdingProfit = currentValue - holdingCost`
- **收益率计算**：`holdingProfitPercent = (holdingProfit / holdingCost) × 100%`
- **今日收益计算**：`estimatedTodayProfit = (currentPrice - yesterdayPrice) × shares`
- **今日涨跌计算**：使用API提供的涨跌幅或计算值

## 6. 异常处理

### 6.1 网络异常
- **API请求失败**：
  - 使用缓存数据
  - 缓存失效时使用默认值（价格=1.0，涨跌幅=0%）
  - 控制台记录错误信息，不中断用户操作

### 6.2 数据异常
- **基金代码无效**：
  - 添加时验证基金代码格式
  - API返回404时提示用户检查基金代码
- **数据格式错误**：
  - 使用默认值填充缺失字段
  - 数值类型转换时处理异常情况

### 6.3 存储异常
- **localStorage 存储失败**：
  - 静默处理，不影响核心功能
  - 控制台记录警告信息

## 9. 附录

### 9.1 术语定义
- **持仓金额**：用户输入的当前市值（包含收益），即用户实际看到的持仓总金额
- **持有成本**：用户投入的资金总额（持仓成本价 × 持有份额）
- **持仓市值**：当前基金价格 × 持有份额
- **持有收益**：持仓市值 - 持仓成本
- **持有收益率**：持有收益 / 持仓成本 × 100%
- **估算今日收益**：基于当日价格变化估算的收益
- **估算今日涨跌**：当日基金净值的涨跌幅

### 9.2 计算公式汇总
```
持有成本 = 持仓份额 × 持有均价（元/份）
当前市值 = API 当前净值 × 持有份额（由接口获取）
持有收益 = 当前市值 - 持有成本
持有收益率 = (持有收益 / 持有成本) × 100%
估算今日收益 = (当前价格 - 昨日净值) × 持有份额
估算今日涨跌 = API涨跌幅 或 ((当前价格 - 昨日净值) / 昨日净值) × 100%
```

### 9.3 数据字段说明
- **基金代码**：6位数字，如 "000001"
- **基金名称**：从 API 自动获取
- **持仓份额**：用户输入的持仓份额（份）
- **持有均价**：用户输入的成本价（元/份）
- **当前市值 / 持仓金额**：由 API 净值 × 持有份额 计算得出
- **当前价格**：从 API 获取（本交易日结束后取最新公布净值）
- **昨日净值**：从 API 获取
- **涨跌幅**：从 API 获取或计算得出
